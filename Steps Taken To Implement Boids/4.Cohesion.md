The second rule we have to implement to get the Boids somewhat flying together is Cohesion, in which the rule states:
- Each Boid must steer to move towards the average position, (center of mass), of local flockmates

To implemenet this, we will take some very similar steps similar to the Seperation rule, and I'll later, hopefully try to modify the rules into one loop, (you'll see as we begin :)

1. The first step is to create a new PVector called center in a new void function called Cohesion, as for Cohesion we will be trying to get the center mass of all nearby Boids, this Pvector will also be in the Boids class.
2. Now we create a int value called TotalNumOfBoidsInRadius, which literally just tracks the number of Boids in its Cohesion radius, but if you've picked it up, what Cohesion Radius?
3. In our setUp function called Boid, we create a variable called CohesionRadius and also initialize that variable to a radius of 100 in my case.
4. Now that we have our radius size set, we go back to the Boid class and loop through all the Boids, comparing their distance between our Boid and the other Boid to the Cohesion Radius.
5. If the Cohesion Radius is larger then the distance between the two Boids, (meaning the Boid is within the Cohesion Radius), and the Boid we are checking in not this Boid, (Making sure the current Boid is not the Boid we are looping over), then we add the position of that Boid to our PVector center variable.
6. Lastly we increase the TotalNumOfBoidsInRadius tracker by one.
7. Now outside of the for loop, we check if the TotalNumOfBoids equals zero, if so we just don't try to adjust the following variables of the current Boid.
8. Next we divide the center PVector varaible by the amount of Boids in the radius (center.div(TotalNumOfBoidsInRadius)).
